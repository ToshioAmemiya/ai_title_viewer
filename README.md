# ai_title_viewer

ゴミを含んだファイル名をそのままAIで検索し、  
同時に「使える正規表現」を育てていくためのタイトル観測ツール。

タイトル以外の“ノイズ”は、時代やコミュニティによって変化します。
このツールは、あなた自身の感覚で「残したい名前」「消したいノイズ」を育てるための工房です。
正規表現がわからなくても大丈夫。AIと一緒に、あなたの“名前の美学”を形にしましょう。

---

## これは何か

ai_title_viewer は、ファイル名を直接変更するソフトではありません。  
また、正規表現を自動で完成させるツールでもありません。

このアプリがやっているのは、次の2つです。

- ゴミを含んだファイル名を、そのまま検索に投げること  
- その「ゴミ」が何なのかを観測し、正規表現として言語化していくこと  

多くの整理ツールは、最初に「正しい名前」を決めようとします。  
ai_title_viewer は逆に、**最初は必ず失敗する**ことを前提に設計されています。

だから、整えません。  
まず投げます。  
うまくいかなかった理由を、あとから拾います。

---

## なぜ検索にAIを使うのか

ファイル名には、常に様々な「ゆらぎ」が含まれます。

- 日本語表記のゆらぎ（全角・半角、漢字・ひらがな、略称）
- 記号や装飾のゆらぎ（【】、()、[]、★ など）
- 並び順や付加情報のゆらぎ（作者名、年、版、形式）

これらを最初から正規表現で吸収しようとすると、  
どうしても「正解の形」を先に決める必要があり、作業が止まりがちです。

ai_title_viewer では、まず **AI検索を使って投げてしまう** ことを選びました。

AIは、
- 表記が多少違っていても
- 情報が余計に付いていても

「だいたい何のことか」を理解してくれます。

検索結果が多少ズレても問題ありません。  
ズレた理由こそが、あとで正規表現として拾うべき材料になります。

---

## このツールの2本柱

ai_title_viewer は、次の2つの作業を明確に分けて考えています。

- 検索で情報を集める  
- AIで正規表現を磨いていく  

この2本柱があるからこそ、  
「意味を理解すること」と「ゴミを消すこと」を混同しません。

---

## 正規表現は「材料」を集めて磨きます

ai_title_viewer では、  
様々なパターンのファイル名を集め、  
それらを材料としてAIに渡し、  
「何が共通で、何がノイズか」を考えさせながら  
正規表現を磨いていきます。

単一の例から正規表現を作ると、  
その例に引きずられた式になりがちです。

複数の例を並べることで、  
AIは「消したいもの」と「守りたい形」を  
比較しながら考えることができます。

---

## AIに渡すプロンプトの例

工房では、次のような文章をそのままAIに投げることを想定しています。

以下はファイル名の例です。

【ノイズを含むタイトル】

[ABC] 作品名 第01巻 (修正版).zip

作品名_第1巻_作者名.pdf

【DL版】作品名01(特典付き).rar

作品名 v2 最終版.txt

【ノイズを含まないタイトル（守りたい形）】

作品名 第1巻

作品名 第2巻

これらをもとに、
「ノイズ部分」を除去し、
「守りたい形」を残す正規表現を提案してください。

正規表現が何を消しているのか、
人が読んで分かる説明も添えてください。



このプロンプトは一例です。  
ai_title_viewer は、プロンプトの形を固定しません。

## 作業の流れ

1. ゴミを含んだタイトルをそのまま表示する  
2. 気になったタイトルをそのまま検索に投げる  
3. ノイズを含むタイトルを集める  
4. ノイズを含まないタイトルも集める  
5. 集めた材料をAIに渡す  
6. 正規表現を試し、必要なら捨てる  
7. うまくいったものだけを育てていく  

ai_title_viewer は、  
正解を一気に出すためのツールではありません。

失敗しながら、  
少しずつ「使える形」に近づけていくための道具です。

---

## ジャンルとTMPについて

ジャンルは固定の分類ではありません。  
ユーザーが自由に増やせる「文脈ラベル」です。

TMP は試作専用のスコープです。

- とりあえず貼る  
- 試す  
- 良ければ昇格  
- ダメなら捨てる  

失敗を隔離するために存在します。

---

## 他のソフトとの関係

ai_title_viewer は、実際のファイル名変更を行いません。

生成・育成した正規表現は、  
リネーム専用ソフトなど、他のツールで使うことを前提にしています。

---

## 向いている人

- ファイル名のゴミに毎回悩まされている人  
- 正規表現は書けるが、育てるのが大変な人  
- AIを魔法ではなく、補助脳として使いたい人  

## 向いていない人

- ワンクリックで完全自動化したい人  
- ファイル名を即座に変更したい人  
- 正規表現を意識せずに済ませたい人  

---

JSONでの書き出しについて
ai_title_viewer では、作業内容を JSON 形式で書き出すことができます。

この JSON には、次の情報が含まれます。

- 収集したタイトル例（ノイズを含む／含まない）
- 無視語
- 正規表現ルール
- ルールの有効／無効状態
- TMP・共通・ジャンル別といった適用スコープ


### なぜ JSON なのか

JSON で書き出す目的は、バックアップだけではありません。

- 他の環境で再利用する
- 別のジャンルに流用する
- 他人とルールを共有する
- 将来、別のツールで読み込む

ai_title_viewer は、
作業結果をアプリ内に閉じ込めないことを前提にしています。


### JSON は「成果物」です

このツールにおいて、
JSON は中間データではなく「成果物」です。

時間をかけて育てた正規表現や無視語は、
JSON として残り、
他のリネームソフトやスクリプトで使うことができます。

ai_title_viewer 自体は、
それらを作るための作業場に過ぎません。

### 共有を前提にした設計

JSON は、人が読める構造を保つように設計されています。

- どの正規表現が
- どのジャンルで
- 何を消しているのか

後から見返したり、
他人が受け取ったときにも理解できることを重視しています。

※ Windows向け配布版では、
本体と工房は1つの実行ファイルにまとめられています。


### 初期状態に戻したい場合

配布されている `workspace_default.json` を読み込むことで、
無視語・正規表現・ジャンル設定を含めた初期状態に戻すことができます。

この JSON は「安全な初期値」として設計されています。
